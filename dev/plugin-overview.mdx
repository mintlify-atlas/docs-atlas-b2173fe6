System Informer loads plugins at startup from a `plugins/` subdirectory next to `SystemInformer.exe`. Each plugin is a Windows DLL that registers itself with the host and then subscribes to the callbacks it cares about. Plugins can add columns to any of the main tree views, inject items into context menus and the main menu, append property pages to process and service dialogs, contribute sections to the System Information and mini-info panels, and register notification-area tray icons.

## How plugins are loaded

<Steps>
  <Step title="Create the plugins directory">
    Create a directory named `plugins` in the same folder as `SystemInformer.exe` and copy your plugin DLL into it.
  </Step>
  <Step title="Enable plugins in Options">
    Open **Options â†’ General** and make sure the **Enable plugins** checkbox is ticked, then restart System Informer.
  </Step>
  <Step title="Verify loading">
    Go to **Help â†’ Plugins** to confirm your plugin appears in the list. If it does not, check the startup error log â€” the host reports load failures there.
  </Step>
</Steps>

<Note>
  System Informer only loads plugins when the executable is named `SystemInformer.exe`. Renaming the binary will prevent plugin loading entirely.
</Note>

## Security model

Plugins that are not signed by a trusted publisher receive restricted access to the kernel driver (`ksi.dll`). Unsigned plugins can still use all public `phlib` and `phapp` APIs, but elevated kernel operations will be blocked. Sign your plugin DLL with an Authenticode certificate if you require unrestricted kernel driver access.

## What plugins can do

Plugins hook into the application through a callback registration mechanism. The most common extension points are:

| Extension point | Callback to register |
|---|---|
| Columns in the process, service, or network tree | `GeneralCallbackProcessTreeNewInitializing` / `GeneralCallbackNetworkTreeNewInitializing` |
| Process context-menu items | `GeneralCallbackProcessMenuInitializing` |
| Main menu entries (Tools submenu, etc.) | `GeneralCallbackMainMenuInitializing` |
| Property pages for processes | `GeneralCallbackProcessPropertiesInitializing` |
| Property pages for services | `GeneralCallbackServicePropertiesInitializing` |
| System Information sections | `GeneralCallbackSystemInformationInitializing` |
| Tray icons | `GeneralCallbackTrayIconsInitializing` |

See [Plugin callbacks](/dev/plugin-callbacks) for the complete reference and [Plugin examples](/dev/plugin-examples) for working code.

## Bundled plugins

The following plugins ship in every release of System Informer and live in the `plugins/` subdirectory.

  </Card>
  <Card title="DotNetTools" icon="code">
    .NET CLR assembly inspection (loaded assemblies and app domains), CLR performance counters, and enhanced thread-stack symbol resolution for managed frames.

  </Card>
  <Card title="UserNotes" icon="note-sticky">

  </Card>
  <Card title="ExtendedNotifications" icon="bell">
