title: "Plugin callbacks"
description: "Complete reference for every general and plugin-lifecycle callback in System Informer, including parameter types and thread context."
---

The callback system is how your plugin receives events from System Informer. There are two independent enumerations:

- **`PH_GENERAL_CALLBACK`** — system-wide events broadcast to all registered listeners.
- **`PH_PLUGIN_CALLBACK`** — events delivered only to a specific plugin instance.

## Registering a callback

Use `PhRegisterCallback` for both kinds:

```c
// General callback (broadcast to all plugins):
PhRegisterCallback(
    PhGetGeneralCallback(GeneralCallbackProcessesUpdated),
    MyProcessesUpdatedHandler,
    NULL,                                   // optional context passed to handler
    &MyProcessesUpdatedRegistration         // PH_CALLBACK_REGISTRATION, must stay alive
    );

// Plugin lifecycle callback (only your plugin):
PhRegisterCallback(
    PhGetPluginCallback(PluginInstance, PluginCallbackLoad),
    MyLoadHandler,
    NULL,
    &MyLoadRegistration
    );
```

Every callback handler has the signature:

```c
_Function_class_(PH_CALLBACK_FUNCTION)
VOID NTAPI MyHandler(
    _In_opt_ PVOID Parameter,
    _In_opt_ PVOID Context
    );
```

`Parameter` carries the event-specific data described below. `Context` is whatever you passed as the third argument to `PhRegisterCallback`.

<Tip>
  Declare every `PH_CALLBACK_REGISTRATION` as a global or static variable. The registration must remain valid until the plugin is unloaded — passing a stack-allocated registration will corrupt memory when the stack frame is released.
</Tip>

## General callbacks

<Accordion title="Window and lifecycle events">

### `GeneralCallbackMainWindowShowing` — `0`
**Thread:** main thread  
**Parameter:** `INT ShowCommand` (the `nCmdShow` value passed to `WinMain`)

Fired when the main window is about to be shown for the first time. This is the right place to create toolbar controls, register layout padding callbacks, and perform other one-time UI setup. ExtendedTools calls `EtInitializeDiskTab`, `EtRegisterToolbarGraphs`, and `EtFramesMonitorStart` here.

---

### `GeneralCallbackMainWindowTabChanged` — `28`
**Thread:** main thread  
**Parameter:** `INT NewIndex` (zero-based tab index)

Fired whenever the user switches between the Processes, Services, and Network tabs.

---

### `GeneralCallbackOptionsWindowInitializing` — `34`
**Thread:** main thread  
**Parameter:** `PPH_PLUGIN_OPTIONS_POINTERS`

Called when the Options window is opening. Use the `CreateSection` function pointer in the parameter to add your own options page:

```c
_Function_class_(PH_CALLBACK_FUNCTION)
VOID NTAPI ShowOptionsCallback(
    _In_ PVOID Parameter,
    _In_opt_ PVOID Context
    )
{
    PPH_PLUGIN_OPTIONS_POINTERS optionsEntry = (PPH_PLUGIN_OPTIONS_POINTERS)Parameter;

    optionsEntry->CreateSection(
        L"NetworkTools",
        PluginInstance->DllBase,
        MAKEINTRESOURCE(IDD_OPTIONS),
        OptionsDlgProc,
        NULL
        );
}
```

*(Source: `plugins/NetworkTools/main.c`)*

---

### `GeneralCallbackSettingsUpdated` — `85`
**Thread:** main thread  
**Parameter:** none

Fired after the user closes the Options dialog with OK, or whenever settings are saved programmatically. Re-read any cached settings here.

</Accordion>

<Accordion title="Process provider events">

### `GeneralCallbackProcessesUpdated` — `1`
**Thread:** main thread  
**Parameter:** `ULONG RunId` (cast from `PVOID` via `PtrToUlong`)

Broadcast after every process provider refresh cycle. `RunId` starts at 1 and increments with each cycle. Many plugins skip early cycles:

```c
_Function_class_(PH_CALLBACK_FUNCTION)
VOID NTAPI ProcessesUpdatedCallback(
    _In_opt_ PVOID Parameter,
    _In_opt_ PVOID Context
    )
{
    ProcessesUpdatedCount = PtrToUlong(Parameter);
}
```

*(Source: `plugins/ExtendedTools/main.c`)*

---

Fired when a new process is detected by the provider.

<Accordion title="Menu initializing callbacks">

All menu callbacks pass `PPH_PLUGIN_MENU_INFORMATION`:

```c
typedef struct _PH_PLUGIN_MENU_INFORMATION
{
    PPH_EMENU Menu;
    HWND OwnerWindow;
    union
    {
        struct { ULONG SubMenuIndex; } MainMenu;
        struct { PPH_PROCESS_ITEM *Processes; ULONG NumberOfProcesses; } Process;
        struct { PPH_SERVICE_ITEM *Services; ULONG NumberOfServices; } Service;
        struct { PPH_NETWORK_ITEM *NetworkItems; ULONG NumberOfNetworkItems; } Network;
        struct { HANDLE ProcessId; PPH_THREAD_ITEM *Threads; ULONG NumberOfThreads; } Thread;
        struct { HANDLE ProcessId; PPH_MODULE_ITEM *Modules; ULONG NumberOfModules; } Module;
        struct { HANDLE ProcessId; PPH_MEMORY_NODE *MemoryNodes; ULONG NumberOfMemoryNodes; } Memory;
        struct { HANDLE ProcessId; PPH_HANDLE_ITEM *Handles; ULONG NumberOfHandles; } Handle;
        struct { PPH_STRINGREF SectionName; PPH_PROCESS_GROUP ProcessGroup; } MiListSection;
    } u;
    ULONG Flags;
    PPH_LIST PluginHookList;
} PH_PLUGIN_MENU_INFORMATION, *PPH_PLUGIN_MENU_INFORMATION;
```

Use `PhPluginCreateEMenuItem` to create items and `PhInsertEMenuItem` to attach them to the menu. The `PluginCallbackMenuItem` plugin callback fires when the user selects one of your items.

## Plugin lifecycle callbacks

These are registered with `PhGetPluginCallback(PluginInstance, ...)` and are delivered only to your plugin.

<Accordion title="Plugin lifecycle callbacks">

### `PluginCallbackLoad` — `0`
**Thread:** main thread  
**Parameter:** `PPH_LIST` of startup parameters (may be `NULL`)

Called after the host has finished loading all plugins. Read your settings and perform any initialisation that does not require the main window here.

```c
_Function_class_(PH_CALLBACK_FUNCTION)
VOID NTAPI LoadCallback(
    _In_opt_ PVOID Parameter,
    _In_opt_ PVOID Context
    )
{
    GeoLiteDatabaseType = PhGetIntegerSetting(SETTING_NAME_GEOLITE_DB_TYPE);
    NetworkExtensionEnabled = !!PhGetIntegerSetting(SETTING_NAME_EXTENDED_TCP_STATS);
}
```

*(Source: `plugins/NetworkTools/main.c`)*

---

### `PluginCallbackUnload` — `1`
**Thread:** main thread  
**Parameter:** `BOOLEAN SessionEnding` (cast from `PVOID` via `PtrToUlong`)

Called just before the plugin is unloaded. If `SessionEnding` is `TRUE`, Windows is shutting down or the user is logging off; skip expensive teardown in that case.

```c
_Function_class_(PH_CALLBACK_FUNCTION)
VOID NTAPI UnloadCallback(
    _In_ PVOID Parameter,
    _In_ PVOID Context
    )
{
    BOOLEAN SessionEnding = (BOOLEAN)PtrToUlong(Parameter);

    if (SessionEnding)
        return;

    EtEtwStatisticsUninitialization();
    EtFramesMonitorUninitialization();
}
```

*(Source: `plugins/ExtendedTools/main.c`)*

### `PluginCallbackMenuItem` — `3`
**Thread:** main thread or properties thread  
**Parameter:** `PPH_PLUGIN_MENU_ITEM`

```c
typedef struct _PH_PLUGIN_MENU_ITEM
{
    PPH_PLUGIN Plugin;
    ULONG Id;           // the Id you passed to PhPluginCreateEMenuItem
    ULONG Reserved1;
    PVOID Context;      // the Context you passed to PhPluginCreateEMenuItem
    HWND OwnerWindow;   // valid only when the item is chosen
    PVOID Reserved2;
    PVOID Reserved3;
    PPH_PLUGIN_MENU_ITEM_DELETE_FUNCTION DeleteFunction;
} PH_PLUGIN_MENU_ITEM, *PPH_PLUGIN_MENU_ITEM;
```

Dispatch on `menuItem->Id` to determine which item was chosen:

```c
_Function_class_(PH_CALLBACK_FUNCTION)
VOID NTAPI MenuItemCallback(
    _In_ PVOID Parameter,
    _In_opt_ PVOID Context
    )
{
    PPH_PLUGIN_MENU_ITEM menuItem = (PPH_PLUGIN_MENU_ITEM)Parameter;

    switch (menuItem->Id)
    {
    case NETWORK_ACTION_PING:
        ShowPingWindow(menuItem->OwnerWindow, (PPH_NETWORK_ITEM)menuItem->Context);
        break;
    case NETWORK_ACTION_TRACEROUTE:
        ShowTracertWindow(menuItem->OwnerWindow, (PPH_NETWORK_ITEM)menuItem->Context);
        break;
    case NETWORK_ACTION_WHOIS:
        ShowWhoisWindow(menuItem->OwnerWindow, (PPH_NETWORK_ITEM)menuItem->Context);
        break;
    }
}
```

*(Source: `plugins/NetworkTools/main.c`)*
